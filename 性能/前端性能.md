一、资源加载与优化

1. 压缩与精简
  - 代码压缩：uglify本地压缩、gzip服务端压缩；
  - 图片优化：使用体积小的图片格式Webp、TinyPNG压缩图片体积

2. 减少请求与利用缓存
  - Tree Shaking：利用Webpack等构建工具，移除未使用的js和css代码，webpack-bundle-analyzer
  - Code Splitting：利用Webpack等构建工具，将代码按照路由或组件拆分成多个chunk，webpack-bundle-analyzer
  - HTTP缓存：为静态的资源（图片、CSS、JS）设置合适的Cache-Control和Etag头，利用浏览器缓存

3. 网络优化
  - CDN：静态资源分发，用户从最近的服务器节点获取资源
  - 预加载：预加载关键资源，需谨慎使用避免阻塞关键资源，影响首屏加载的速度

二、渲染优化 

1. 关键渲染路径优化：
  - CSS：将关键CSS内嵌到<head>中，减少首次渲染的RTT（Round Trip Time 来回通信的往返延迟时间）
  - JavaScript：非关键信息，可以异步操作，避免阻塞HTML的解析，减少主线程的工作量

2. 代码的执行效率：
  - 尽量避免样式重排，这也会导致浏览器反复计算布局
  - 防抖与节流

3. 感知性能与用户体验：
  - 骨架屏，loading动画：减少用户等待的焦虑感
